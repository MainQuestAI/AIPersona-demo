## 截图区域标注

截图展示的是：一个已完成的 study 的 workbench 页面（`/studies/{id}/workbench`）

## 一、全局 Shell + Header

### H-01：Header 标题仍然是英文 "Study Detail"
- **位置**：`app-shell.tsx:14` — `getRouteMeta` 对 `/studies/` 路径硬编码返回 `{ label: 'Study Detail' }`
- **问题**：其他页面的 label 已改中文（业务总览/研究项目/孪生中心），但 study detail 路径的 label 没改
- **修复**：`getRouteMeta` 返回 `{ label: '研究详情' }`

### H-02：Header 前缀仍然是 "AI Consumer"
- **位置**：`app-shell.tsx:79` — `<div className="eyebrow text-tertiary">AI Consumer</div>`
- **问题**：应该是产品名或品牌名，不是技术内部命名
- **修复**：改为 `AIpersona` 或直接去掉，只保留页面标题

### H-03：Header sub-bar 的 Workbench/Compare/Twins 与 study-detail-layout 内部重复
- **位置**：`app-shell.tsx:96-115` 渲染了 `STUDY_DETAIL_VIEWS` 的 NavLink；`study-detail-layout.tsx:133-149` 又渲染了一遍相同的 NavLink
- **问题**：截图中可以看到 Header 和内容区各有一排 Workbench/Compare/Twins 按钮
- **修复**：二选一。建议保留 Header sub-bar 中的，移除 study-detail-layout 中的

### H-04：STUDY_DETAIL_VIEWS 的 label 仍为英文
- **位置**：`route.ts:91-107` — `label: 'Workbench'`, `'Compare'`, `'Twins'`
- **问题**：和已经中文化的 APP_ROUTES label 不一致
- **修复**：改为 `'研究工作台'`, `'概念对比'`, `'孪生溯源'`

## 二、Rail 侧边栏

### R-01：Rail 展开态上面是中文、下面是英文
- **位置**：`global-rail.tsx:67-74` — 展开态渲染 `{route.label}` 上面 + `{route.railLabel}` 下面
- **问题**：现在 `label` 是中文（业务总览），`railLabel` 是英文（Dashboard）。展开态变成中上英下。折叠态只有图标 + title 属性（用的是 `route.label` 即中文）
- **修复**：两种方案——
  - A）Rail 展开态只显示中文 label，去掉 railLabel 的 eyebrow 行
  - B）railLabel 也改为中文副标题（如"总览"/"项目"/"资产"）

## 三、Study Detail Layout（截图上半部分的大面板）

### SD-01：eyebrow 标签 "Study Detail" 英文
- **位置**：`study-detail-layout.tsx:86` — `<div className="eyebrow text-accent">Study Detail</div>`
- **修复**：改为 `研究详情`

### SD-02：UUID 赤裸暴露在页面上
- **位置**：`study-detail-layout.tsx:107-109` — 直接渲染 `{studyId}`
- **问题**：`8acf6eba-a266-432c-a184-55192cb6cab5` 对用户毫无意义
- **修复**：删除这个 badge，或改为 `Study #1`（用序号）

### SD-03："预算 120.00" 含义不明
- **位置**：`study-detail-layout.tsx:102-104` — 渲染 `decisionSnapshot.costLabel`
- **问题**：来自 `formatCostLabel` 函数，显示 `预算 120.00`——这是 `estimated_cost` 字段，但用户不知道单位（元？美元？API credits？）
- **修复**：如果数据不准确就不显示，或加单位说明

### SD-04：Evidence Chain 4 卡片占屏幕过多
- **位置**：`study-detail-layout.tsx:151-166` — `lg:grid-cols-4` 的 evidence chain cards
- **问题**：4 个卡片每个内容只有标题+两行描述+一个链接，但占据了整个屏幕宽度的大面板。和下方 workbench 内容区竞争纵向空间
- **修复**：
  - 方案 A：将 study-detail-layout 的信息面板改为可折叠（默认折叠，只显示标题+状态）
  - 方案 B：移除 evidence chain cards，这些功能已在 workbench 的 result panel 中提供

### SD-05：study-detail-layout 是额外包装层，导致 workbench 被压缩
- **位置**：`routes.tsx:56-74` — StudyDetailLayout 作为 wrapper 渲染 Outlet
- **问题**：Layout 自己渲染了一大块"研究概览"面板（包括标题、badges、下一步建议、运行消耗、证据覆盖、view tabs、evidence chain），然后下面才是 workbench 的 3 栏布局。workbench 的对话区被挤到剩余空间里
- **这是截图中对话区几乎为空的根本原因**：不是没有内容，是被上面的面板压扁了
- **修复**：study-detail-layout 应该极简——只保留一行 breadcrumb + view tabs，不渲染 decision snapshot 和 evidence chain

## 四、Workbench 对话区

### W-01：对话区几乎为空
- **位置**：截图中左下角区域
- **问题**：对于旧的非 agent 流程创建的 study，`study_message` 表中没有数据。新的 `WorkbenchPage` 只从 `/agent/messages` 拉取消息，拉到空列表。旧的 `buildConversationEventsFromArtifacts` fallback 已被移除
- **修复**：当 agent messages 为空时，应 fallback 到从 artifacts 构建的 conversation events（保留旧逻辑作为兼容）

### W-02：对话区只显示了研究问题标题 + "已完成" badge + 一行摘要
- **位置**：workbench-page 的 header bar 部分
- **问题**：这些信息和 study-detail-layout 的信息完全重复——用户在同一屏看到两次"建议推进 清泉+"
- **修复**：workbench 内部的 header bar 应该在 study-detail-layout 存在时简化或隐藏

### W-03：右侧 ResultPanel 内容正确但和 study-detail-layout 重复
- **问题**：ResultPanel 显示推荐结论、AI 综合评估、研究结论 tab——但 study-detail-layout 已经显示了同样的 decision snapshot
- **修复**：study-detail-layout 精简后，ResultPanel 成为唯一的结果展示区

## 五、对话式 Agent 体验缺失

### A-01：新创建研究没有自动启动 Agent
- **位置**：`workbench-placeholder.tsx:374` — `navigate('/workbench/{id}?playback=1')` 但不再有 playback；也没有调用 `startAgent`
- **问题**：创建研究后跳转到 workbench，但 agent 没启动，对话区是空的
- **修复**：`handleCreateDemoStudy` 成功后，应调用 `startAgent(studyId)` 启动 agent 对话

### A-02：Studies 页的"创建 Demo Study"没接入 Agent 流程
- **位置**：`studies-page.tsx:43-55` — `handleCreate` 调用 `createDemoStudy()` 后直接 navigate，没有调用 `startAgent`
- **修复**：创建后也需要 `startAgent`

### A-03：旧 study 回访时 agent 对话为空
- **问题**：已完成的 study 没有 `study_message` 数据，workbench 对话区完全空白
- **修复**：当 agent messages 为空但 artifacts 存在时，应 fallback 到 artifact-based conversation events
